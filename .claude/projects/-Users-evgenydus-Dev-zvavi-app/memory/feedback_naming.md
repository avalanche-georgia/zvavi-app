---
name: Explicit variable naming
description: Always use explicit, descriptive variable names — never single-letter or abbreviated names
type: feedback
---

Always use explicit, descriptive variable names. Never use single-letter names (e.g. `p`, `t`, `e`) or overly short abbreviations.

**Why:** Code readability and consistency — the team prefers names that clearly communicate intent.

**How to apply:** In all code: loop variables, function parameters, destructured values, callbacks, etc. For example, use `partner` not `p`, `error` not `e`, `event` not `e`.
